# TLM AI Chat Cache 使用教程

## 这个模组做了什么？

车万女仆（Touhou Little Maid）的 AI 对话功能需要调用 LLM（大语言模型）来将你说的话翻译成游戏操作。每次对话都要等待网络请求，既慢又花钱。

**TLM AI Chat Cache** 在本地建立一个"你说的话 → 对应操作"的映射缓存。当你说过的话再次出现时，直接从缓存读取，**零延迟响应、零 API 费用**。说得越多，缓存越聪明，对 LLM 的依赖越少。

模组还内置了 70+ 条常见中英文指令的预置映射，装上就能用，不需要任何 LLM 配置就能处理大部分常见命令。

---

## 安装

### 前置要求

- Minecraft 1.20.1
- Forge 47.2.0 或更高版本
- [Touhou Little Maid](https://github.com/TartaricAcid/TouhouLittleMaid) 1.5.0 或更高版本

### 安装步骤

1. 下载 `tlm-ai-cache-1.0.0.jar`
2. 放入游戏的 `mods/` 文件夹
3. 启动游戏，模组会自动生效

首次启动后，模组会在 `config/tlm-ai-cache/` 目录下生成配置文件。

## 基本使用

### 直接对话即可

和女仆正常对话就行，不需要任何额外操作。模组在后台自动工作：

1. **你对女仆说话**（比如"去攻击"）
2. **模组自动匹配缓存** — 如果你说的话匹配到了已有映射，女仆立刻执行对应操作
3. **没匹配到？交给 LLM** — 如果缓存里没有，请求照常发给 LLM 处理

### 内置指令示例

以下指令开箱即用，无需 LLM：

| 你可以说 | 女仆会做 |
|---------|---------|
| 攻击、打怪、战斗 | 切换到攻击模式 |
| attack、fight、combat | 切换到攻击模式 |
| 种地、种菜、种田 | 切换到种植模式 |
| farm、plant、grow | 切换到种植模式 |
| 跟着我、跟随、过来 | 跟随玩家 |
| follow、come、follow me | 跟随玩家 |
| 待命、别动、等着、站着 | 原地待命 |
| stay、wait、stop | 原地待命 |
| 剪羊毛 | 切换到剪羊毛模式 |
| 挤牛奶、挤奶 | 切换到挤奶模式 |
| 喂食、喂动物 | 切换到喂食模式 |
| 钓鱼 | 切换到钓鱼模式 |
| 放火把、插火把 | 切换到放火把模式 |
| 灭火 | 切换到灭火模式 |
| 割草 | 切换到割草模式 |
| 采蜜 | 切换到采蜜模式 |
| 休息、空闲 | 切换到空闲模式 |

模组会自动处理语气词和多余的修饰语。比如"你能帮我去种一下地吗？"会被归一化为"种地"，依然能命中缓存。

### 学习新指令

当你说了一句缓存里没有的话，并且 LLM 返回了结果后，模组会弹出一条确认消息：

```
[女仆名] 我理解为【种植模式】 [✔ 确认] [✘ 不对]
```

- 点击 **[✔ 确认]**：这句话和对应的操作会被记住，下次说同样的话就不用再调 LLM 了
- 点击 **[✘ 不对]**：会展示所有可用的操作列表，你点选正确的操作，模组会记住正确的映射
- **不点击**：30 秒后自动超时，本次不缓存，操作照常执行

学到的映射会保存在 `config/tlm-ai-cache/learned.json`，重启游戏不会丢失。

## 管理命令

所有命令以 `/tlmcache` 开头。

### 查看缓存

```
/tlmcache list
/tlmcache list 2
```

分页列出所有缓存条目（每页 10 条），按命中次数排序。可以指定页码。

### 查看统计

```
/tlmcache stats
```

显示缓存统计信息：总条目数、内置条目数、学习条目数、命中率、最常用的 5 个映射。

### 删除条目

```
/tlmcache remove <key>
```

删除指定的缓存条目。`<key>` 是归一化后的文本（可以通过 `/tlmcache list` 查看）。需要 OP 权限。

### 清空学习缓存

```
/tlmcache clear
```

清空所有玩家学习到的缓存映射，不影响内置的预置映射。需要 OP 权限。

### 重新加载

```
/tlmcache reload
```

从磁盘文件重新加载缓存。手动编辑了 JSON 文件后可以用这个命令刷新。需要 OP 权限。

### 导出与导入

```
/tlmcache export
/tlmcache import <文件名>
```

- **导出**：将学习到的缓存保存到 `config/tlm-ai-cache/learned.json`
- **导入**：从指定的 JSON 文件导入缓存条目，适合社区之间共享词库

均需要 OP 权限。

## 配置

配置文件在游戏启动后自动生成，可以通过 Forge 的 Mod 配置界面修改，也可以直接编辑配置文件。

### 基础设置

| 配置项 | 默认值 | 说明 |
|-------|-------|------|
| `enable_cache` | `true` | 总开关。设为 `false` 则完全禁用缓存，所有对话直接走 LLM |
| `require_confirmation` | `true` | 是否需要玩家确认才将新映射写入缓存。设为 `false` 则 LLM 返回的结果自动写入缓存 |
| `shared_cache` | `true` | 多人服务器中是否共享缓存。`true` 表示所有玩家共享同一份缓存 |
| `op_only_confirm` | `false` | 是否只允许 OP 确认新的缓存映射。在公共服务器上建议开启，防止恶意教学 |

### 归一化设置

| 配置项 | 默认值 | 说明 |
|-------|-------|------|
| `extra_cn_stopwords` | `""` | 自定义额外的中文停用词，用逗号分隔。这些词在归一化时会被移除 |
| `extra_en_stopwords` | `""` | 自定义额外的英文停用词，用逗号分隔 |

### 调试设置

| 配置项 | 默认值 | 说明 |
|-------|-------|------|
| `show_cache_debug` | `false` | 在聊天栏显示缓存命中/未命中的调试信息，便于排查问题 |

---

## 多人服务器注意事项

- **共享缓存**（默认开启）：所有玩家的学习结果共享，一个人教过的指令其他人也能用
- **OP 确认**：在公共服务器上建议开启 `op_only_confirm`，防止玩家故意教错误的映射（比如把"攻击"映射到"跟随"）
- 管理命令（`clear`、`remove`、`reload`、`export`、`import`）需要 OP 权限

## 文件说明

模组的数据文件位于 `config/tlm-ai-cache/` 目录下：

| 文件 | 说明 |
|-----|------|
| `builtin.json` | 内置预置映射，随模组版本更新。不要手动修改，更新模组后会被覆盖 |
| `learned.json` | 玩家学习到的映射，不会被模组更新覆盖。可以手动编辑后用 `/tlmcache reload` 刷新 |

`learned.json` 中的映射优先级高于 `builtin.json`，如果同一个 key 同时存在于两个文件中，以 `learned.json` 为准。

## 常见问题

### Q: 不配置 LLM 也能用吗？

可以。模组内置了 70+ 条常见指令的映射，覆盖了大部分日常操作。不配置 LLM 的情况下，只要你说的话能匹配到内置映射，女仆就能正确响应。只有缓存未命中时才需要 LLM。

### Q: 卸载这个模组会影响车万女仆吗？

不会。这个模组是纯增量的缓存层，不修改车万女仆的任何原有数据。卸载后女仆的 AI 对话功能完全恢复原样。

### Q: 女仆没有反应怎么办？

1. 确认模组已正确安装（检查 Mod 列表）
2. 用 `/tlmcache stats` 查看缓存是否正常加载
3. 开启调试模式：将 `show_cache_debug` 设为 `true`，查看缓存是命中还是未命中
4. 用 `/tlmcache list` 检查缓存内容是否正确

### Q: 怎么修正一个错误的缓存映射？

用 `/tlmcache remove <key>` 删除错误的条目，然后重新对女仆说一次，通过确认 UI 建立正确的映射。

### Q: 可以和朋友分享我的词库吗？

可以。用 `/tlmcache export` 导出你的 `learned.json`，把文件发给朋友，朋友用 `/tlmcache import` 导入即可。
